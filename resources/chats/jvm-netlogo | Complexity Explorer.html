Duncan McGreggor 3:40 PM
for starters, this might add some missing context: Java, as a language, was originally created to be able to run on different architectures (Sun/Solaris, UNIX, Windows, VMS, later Linux, etc.) ... mostly because coders in the 70s and 80s spent soooo much of their time porting a critical program from one machine architecture to another  (e.g., from a program on a PDP-11 running old, mid-70s UNIX to a DEC Alpha running Tru64 UNIX). In order to make this happen, they decided to implement a "virtual machine" giving the world at large one machine to write for: the Java Virtual Machine ... then they took the responsibility upon themselves to create Java runtimes that would compile/execute/etc on any lots of different architectures
3:41 PM
thus the Java language itself, and therefore programs written on them, wouldn't have to be ported from one arch to another
3:41 PM
once they created this JVM, people started doing interesting things with it
3:41 PM
compiler optimizations was the big focus for about 10 years
3:42 PM
having done that, it became a very attractive platform for language designers
3:43 PM
thus the current multitude of languages -- or conversions of languages -- that now run on the JVM

Duncan McGreggor 11:45 AM
it makes you more efficient for several of reasons:
1) at an almost unthinking/habit level, it provides a structured, disciplined approach/methodology for progressing through a project from beginning to end (this has a name in the industry: "Test Driven Development", or TDD)
2) at a higher level, you can check your logic, and make sure your design and your actual implementation are aiming for and achieving the same things
3) a little further up the conceptual "How", it lets you establish confidence in what you're writing ... avoiding the common pitfall of loosing sight of your project ... or losing a sense of direction in the code
4) at perhaps the highest level (and this is the real secret), long practice of this approach leads to growing clarity ... and eventually extreme clarity ... of mind (at least, with regard to problem solving in the logical world of computing machines!)
11:46 AM
that clarity of mind will eventually result in the ability to look at a problem and, before you even write any code, have the whole thing unfurl in your mind, anticipating many of the problems way ahead of time, and design for a good solution even before a line of code is written
