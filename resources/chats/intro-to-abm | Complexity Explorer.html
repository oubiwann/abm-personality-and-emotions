Duncan McGreggor 10:29 AM
Oh, sorry @Poornima and @Andreas Sjöstedt... just saw the comments now from the other day about JVM, Clojure, tests., etc.
10:30 AM
So Netlogo is implemented in Scala (and maybe a bit of Java?) Scala being a JVM language (Java Virtual Machine)
10:30 AM
so when you start up Netlogo on your machine, there's actually a java process that is running it
10:31 AM
incidentally, this is one of the ways you can get integrated Python support in Netlogo: via Jython (Python on the JVM)
10:32 AM
other JVM languages you should be able to use "easily" with Netlogo include Kotlin, Groovy, JRuby, Clojure, etc.
10:32 AM
That's all pretty high-level stuff
10:34 AM
where it gets interesting from my perspective is at the API level Netlogo publishes its Scala API for use by other JVM languages (that's what this link is, that I posted previously: http://ccl.northwestern.edu/netlogo/docs/scaladoc/org/nlogo/, with the links on the right providing API details for its various packages, etc.)
ccl.northwestern.edu
NetLogo 6.1.0 - org.nlogo
NetLogo 6.1.0 - org.nlogo
10:35 AM
in particular, the headless package lets you start up and run a model and its world without running a GUI
10:36 AM
this means that, using their API, you can load and run your models for the classic types of unit and integration testing you'd do in other languages
10:37 AM
Since I'm a Clojure programmer, I can use all the tooling I've got in tons of other projects to help me more effectively build and test my Netlogo models
10:38 AM
I can commit my changes, push them up to Github, and then have TravisCI run my tests to see/track my builds (see here for an example: https://travis-ci.org/oubiwann/intro-abm/builds) (edited) 
10:40 AM
and yeah, in essence, the builtin Clojure library clojure.test is the Clojure world's answer to JUnit, similar in inspiration and nature to what the Python world's builtin unittest library is for that language
